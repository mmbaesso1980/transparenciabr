# M13 · Pre-flight Billing Check + FinOps Guard

**Pontos:** 3 | **Skill:** `transparenciabr-lei` + `maestro-autonomo`
**Origem:** Incidente Lightning dunning de 27/05/2026 — Maestro v1.0 acionou Vertex sem verificar saldo do crédito promocional R$ 5.952, levando a HTTP 403 PERMISSION_DENIED em toda a billing account `01061C-9EC54F-3C6B7B`.

## Contexto (não repetir)

A skill `maestro-autonomo` já define dois caps de FinOps (F5):
- **Soft cap:** R$ 30/h de queima Vertex
- **Hard cap:** R$ 80/h de queima Vertex
- Reset por janela de 1h.

Ambos pressupõem que o **Maestro autônomo** está rodando. Em operações **manuais** (dossiês compilados pelo Comandante, blind tests, deploys, ingestão), os caps **não eram consultados**. A consequência: queima livre até o crédito acabar, daí Lightning dunning automático.

O Comandante destacou: *"só estou devendo pq erramos acionando a vertex sem crédito, mesmo tendo 100% awareness a este fato"*. Erro de raciocínio econômico do agente, não técnico.

## Contrato

Conjunto completo: **pre-flight check antes de operação pesada → estimativa de custo → bloqueio se billing em dunning → log de cada chamada**.

### Componentes

1. **Script `scripts/preflight_billing_check.sh`**
   - Função `preflight_billing_check [project_id] [min_credit_brl]`
   - Verifica: billing habilitado, BA aberta, Vertex ENABLED, ping HTTP em `gemini-2.5-flash` (mais barato)
   - Exit codes: `0` OK · `1` WARN (operador decide) · `2` BLOCK (aborta)

2. **Integração obrigatória em scripts críticos:**
   - `aurora_v3_maestro/deploy/deploy_all.sh` (antes do `gcloud run deploy`)
   - `aurora_v3_maestro/worker/blind_test_paulo_octavio.py` (antes de `--run-vertex`)
   - `manus_office/dossie_v1/scripts/gerar_dossie_v1.py` (antes da geração)
   - `engines/vertex/client.py` (no inicializador do cliente)

3. **FinOps Guard runtime** (extensão futura, fora deste M13)
   - Decorator `@finops_guard(max_brl=N)` em funções que invocam Vertex
   - Trackeia uso via Firestore `maestro_finops_log` em tempo real
   - Aborta com `RuntimeError` se janela 1h ultrapassar hard cap

4. **Documentação:**
   - `docs/postmortem/v1-0-lightning-dunning.md` (postmortem 5-whys do incidente)
   - Lição registrada em `maestro_memory` com tópico `vertex-lightning-dunning`
